# ADR-0069: Holo Route — typed semantic streams: the routing plane. A pipeline is a content-addressed dataflow graph of deterministic κ-transforms whose seams are type-checked by re-derivation before a byte flows, sealed as one self-verifying κ any peer re-runs with no server

**Status:** Proposed — to be witnessed by `tools/holo-route-witness.mjs` (pure Node, joining the gate's LIVE set) and a required `#holo-route` row in `os/etc/conformance.jsonld` (the gate, ADR-024). The runtime ships at `os/usr/lib/holo/holo-route.mjs` (`window.HoloRoute`), is surfaced to every builder through the SDK façade (`os/usr/lib/holo/holo-sdk.js` — new verbs `pipe`/`route`, the dataflow siblings of `build`/`run`/`share`), and runs identically in the browser and in Node. The decision is not done until it is witnessed (ADR-024); this ADR pins the design and carries the build spec. *(House-keeping: numbered `0069` — `0067` and `0068` were claimed by concurrent ADRs (holo-qvac, holo-settle-plasma) during this work; re-grep `docs/adr/` for the max before adding the next, since numbers advance concurrently.)* Composes Holo Link (ADR-0060), Holo Forge (ADR-0051), Holo Q (ADR-0052) and Holo Orchestrate (ADR-0045); the pipeline κ flows into Own (ADR-0053) · Settle (ADR-0048) · Delegate (ADR-0042) · Share-to-Run (ADR-0064). Mints nothing beyond the existing `hosc:` namespace.

**Context.** The substrate already has two ways for objects to combine, and they meet at two different boundaries. **Holo Link (ADR-0060)** composes at the *call* boundary: module A names module B by its κ, the linker wires export→import across isolated linear memories, and the whole thing instantiates as one synchronous combined module — the unit of composition is a *function*. **Holo Orchestrate (ADR-0045)** composes at the *collaboration* boundary: a work receipt records *who* acted — probabilistic agents, each carrying its NANDA identity, AgentTrust reputation and the UCAN it acted under. Neither is the third boundary that real systems are mostly made of: the **data** boundary — a typed value produced by one transform, consumed by the next, flowing through a *pipeline* of deterministic stages. This is the UNIX pipe, and the substrate did not have it.

A pipe is the most-used composition primitive in computing for a reason: `a | b | c` lets a builder assemble a capability out of parts without any part knowing the others exist. But the classical pipe carries only *bytes* — it is untyped (a mismatch is discovered at runtime, deep in stage `c`, or silently corrupts), unverified (you trust each stage ran), and ephemeral (the pipeline is a shell invocation, not a thing you can name, share, own, or pay for). The AI-native-OS literature (Anunix's *Routing Plane* + *Semantic Streams*, which independently re-derived this OS from content-addressing and provenance — the same primitives this substrate is built on) names the upgrade exactly: pipes should be **type-aware**, carry **semantic flows** not raw bytes, and host **pluggable transformations**. The substrate can do strictly better than "type-aware," because it already has the machinery: Holo Link ships a `holo-iface` custom section in which a module's own bytes (its κ) commit to its *interface types*, and Holo Forge/Holo Q make every transform a deterministic, re-derivable κ-transform with a PROV-O receipt. So the seam between two stages need not be merely *checked* — it can be **verified by re-derivation (Law L5)**, and the whole pipeline can be **one self-verifying κ-object** like every other node in the graph. The gap was exact: there was no layer that takes a list of transform κ's, type-checks the seams from their pinned interfaces *before running*, executes them as a deterministic dataflow, and seals the run as a re-derivable object. Holo Route is that layer — the routing plane, native.

**Decision.** Make a dataflow pipeline a first-class content-addressed object, end to end:

1. **A pipeline is a list of κ-stages; the pipeline is itself a κ.** A stage is a κ — a Holo Forge build, a Holo Q model, any Holo App that exposes a `κ → κ` transform — or `{ kappa, budget? }`. `pipe(a).to(b).to(c)` (fluent) and `route([a, b, c], input)` (declarative) both denote the same object: a PROV-O dataflow DAG whose canonical form is `[stageκ, …] ⊕ entry-type`. Its κ commits transitively to every stage's bytes (each stage κ is *in* the pipeline's bytes), so the pipeline's identity is its composition — the self-referential link the substrate runs on (Law L1). No manifest, no registry, no server.

2. **Seams are type-checked by re-derivation, before a byte flows.** The pipeline declares each stage's seam types (`i32 · str · bytes`, growing to the full WIT vocabulary `rec · list · variant`), so the pipeline's κ *commits to every seam* — exactly the way a Holo Link importer's bytes commit to the type of the extern it calls (the consumer commits to the interface). At assembly time `route` walks the chain and verifies each adjacent seam by re-derivation: producer `out` must unify with consumer `in`, **and each stage is re-derived from the store and checked against its pinned bytes** — the declared entry export must exist, and a `str`/`bytes` seam requires the stage to export the isolated-memory ABI (`alloc` + its own `memory`). A mismatch is **refused at assembly** — cheap, deterministic, with a precise `seam[i→i+1] type mismatch — stage[i] emits str, stage[i+1] expects i32` error — never discovered halfway through a run. So a stage cannot satisfy a seam its pinned bytes structurally can't, and the type it is held to lives in the pipeline's own κ. Reading the entry type directly from a stage's *own* `holo-iface` is the additive path once Holo Forge records exported-function types (today it records only extern/import types, the Holo Link surface).

3. **Each stage runs as a verifiable κ-transform; the output is accepted only after re-deriving its κ (Law L5).** A stage executes via `HoloApp.run` (Holo Forge's deterministic RT2 executor, or Holo Q's greedy decode) — the same engine that already proves `κ(source) ⊕ κ(input) → κ(output)` re-derives byte-for-byte. Route persists each stage output to the store and verifies it by re-derivation before handing it to the next stage. **Memoization is Law L3:** an identical `(stageκ ⊕ inputκ)` is an O(1) rebind, so re-running a pipeline after editing one stage recomputes only that stage and everything downstream — the network gets faster as it grows, and a shared stage links exactly once.

4. **Semantic streams, not bytes — with back-pressure.** A seam typed `list<T>` (or a `stream<T>` descriptor, an additive `holo-iface` shape) flows incrementally: the producer emits elements, the consumer pulls them, and a slow consumer back-pressures a fast producer through an async iterator at the boundary. The lift/lower across isolated component memories is exactly Holo Link's `copyComposite`/`fixup` walker applied per element — Route adds no new ABI, it streams over the one Holo Link already proves. A finite transform is just a stream of length one.

5. **The run seals as one self-verifying object.** A completed pipeline mints a PROV-O activity DAG — each stage a `prov:Activity` (`prov:used` input κ → `prov:generated` output κ, via `hosc:tool` stage κ) — sealed to its own `did:holo` (the UOR envelope, ADR-025). To verify a whole pipeline a peer holds only the route κ and the κ-verified stage bytes, re-runs it, and reproduces every seam and the final κ (`verifyDeep`, Law L5). Tamper anywhere — flip a stage byte, swap a stage, forge an intermediate — and the route κ no longer re-derives, so the run is refused. Because the pipeline is now an object, the rest of the stack applies unchanged: it is **shareable** (`#k=` → it runs fullscreen, ADR-0064), **ownable** (a Title, ADR-0053), **settleable** per proven run (ADR-0048), **delegable** (a UCAN scoped to a pipeline, ADR-0042), and **conscience-gated at each stage dispatch** (the constitution's pre-dispatch gate, ADR-0033 / the A57 verdict, so a stage that the conscience blocks halts the flow fail-closed).

**Magical for builders — the seamless surface.** One import, two new verbs, the same mental model as a shell pipe but typed and verifiable:

```js
import { pipe, route, share } from "@hologram/sdk";

// fluent: assemble a capability out of content-addressed parts
const clean   = await build(`str trim(str s) { … } from "…";`);   // a Forge κ
const summary = "did:holo:sha256:…q-model…";                       // a Holo Q κ
const flow    = pipe(clean.kappa).to(summary);                     // type-checked at assembly

const out = await flow.run(rawText);          // runs the dataflow, verifies every seam (Law L5)
share(flow.kappa);                            // holo://κ — the whole pipeline is one shareable object

// declarative + streaming, identical semantics
for await (const item of route([ingest, embed, rank], corpus, { stream: true })) { render(item); }
```

`pipe`/`route` verify each declared seam by re-derivation against the stage's pinned bytes — the entry export exists, a `str` seam's stage exports the isolated-memory ABI, and adjacent types unify — so a wrong connection is caught **before** the run with a human-readable error, not after. Re-running after editing one stage is O(1) for the unchanged prefix. There is no SDK, no install, no server, no glue — the substrate is the pipe. An **agent** gets the same power over MCP: a `holo_route` tool assembles a pipeline by content address, type-checks it, runs it, and returns the self-verifying route κ (verify, don't trust) — and that κ slots into an Orchestrate work receipt as a single deterministic activity.

**Relationship to Link and Orchestrate (the boundary that is actually new).** These three do not overlap; they meet at three different boundaries. **Holo Link** is the *call* boundary — synchronous, in-process, one combined module, the unit is a *function*, values cross via the canonical ABI. **Holo Orchestrate** is the *collaboration* boundary — the actors are *probabilistic agents*, and the receipt records *who* acted, how trustworthy, under what authority. **Holo Route** is the *data* boundary — the actors are *deterministic transforms*, the flow is typed semantic streams, the seams are re-derived, and the run re-derives byte-for-byte. A Route pipeline is the deterministic sibling of an Orchestrate DAG: where Orchestrate proves a collaboration of agents, Route proves a composition of transforms, and a whole pipeline is one `prov:Activity` an Orchestrate DAG can contain. Route uses Link *inside* a stage (a stage may itself be a linked module) and never re-implements it.

**Consequences.** The pipe — computing's most-used composition primitive — becomes a first-class, typed, verifiable, ownable object. Three classes of failure that classical pipes punt on dissolve: **type errors** (a mismatched seam is refused at assembly by re-derivation, not discovered mid-run or silently corrupting); **trust** (you don't trust each stage ran — the pipeline κ re-derives, so a tampered stage or intermediate is refused); and **ephemerality** (a pipeline is no longer a shell invocation but a `holo://κ` you share, fork, own, pay for, and delegate). It composes the catalog rather than extending it: Forge stages, Q stages, Link inside a stage, Orchestrate around it, Own/Settle/Delegate over it — mint nothing. The honest cost and scope: Route inherits Holo Link's current ABI breadth (scalars, strings, byte-buffers, structs, records, lists, variants compose; `option`/`result`/`enum`/`flags` and large-record flattening are the same additive growth path on the shared `holo-iface` walker), so a seam type Link can't yet lift, Route can't yet stream — the mechanism is complete, the vocabulary is breadth-not-depth. A per-stage **budget** (time/compute) reuses Holo UX's tier-aware resource budget (ADR-0062) verbatim — a stage that would exceed it is refused (Anunix's Execution-Cell budgets, folded in with no new mechanism, NIHITO) — and the store's hot-RAM→κ-store→remote tiering is already Law L3, so the *Memory Plane* idea needs no new layer either. What Route adds is exactly one thing the substrate lacked and nothing more: the routing plane that turns a list of κ-transforms into a typed, verifiable, self-verifying dataflow.
