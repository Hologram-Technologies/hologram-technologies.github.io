# ADR-0086: Holo Perception — the orchestrator SEES the whole holospace by READING the κ-graph it already produces, not by screenshotting pixels: every object carries a code-κ and a visual-κ, and the drift between them is the real-time self-improvement signal

**Status:** Accepted (perception layer landed + Node-witnessed 12/12 + verified LIVE in the browser; full-coverage producer wiring + the raster-edge vision specialist pending). Ships at `system/os/usr/lib/holo/q/holo-q-perception.js` (buildless, dependency-free, DOM-free). It gives Holo Mind (ADR-0081) a live, content-addressed view of every κ-object in a holospace: each object has a **code face** (its source/state/compute-output κ, fed by the fabric/orchestrator) and a **visual face** (its rendered-fragment κ, fed by the renderer's `onFinal`), and a change in either emits an O(1) delta tagged with the axis that moved (`code · visual · both`). A whole-scene κ skips an unchanged perception pass instantly; **drift** — code newer than visual (`codeSeq > visualSeq`) — is the real-time feedback signal the self-improvement loop acts on, `coherent` the goal state. Verified live: two prompts in the creator demo → the scene tracks 2 κ-objects, each with BOTH faces, all coherent (`serverless · 2 κ-objects · 2 coherent` on screen); a code edit with no re-render flags drift, which clears when the visual catches up. **Deferred:** registering EVERY producer (widgets, holospace objects, external apps — the renderer auto-covers visuals and the orchestrator auto-covers code; the rest is wiring), and the raster-edge vision specialist (a small on-device VLM via the mux "vision" task, ADR-0084) for non-κ content, plus a `#holo-perception` witness row + os-closure re-lock.

**Context (the load-bearing decision).** The instinct for "give the agent vision over the screen" reaches for the popular computer-use stacks — Open Interpreter, OmniParser, Qwen2.5-VL, UI-TARS, Skyvern. Every one of them solves a problem this substrate does not have: they SCREENSHOT an opaque screen and run a heavy vision model to RECOVER structure from pixels, then act. On Hologram the structure was never lost — every object is a content-addressed κ-object, and the renderer (ADR-0085) already stamps every on-screen fragment with a render-κ. So the orchestrator does not need vision over pixels; it needs to READ the κ-graph it already produces. That is the opposite of screenshotting: structured, deterministic, sub-millisecond, serverless. Evaluated against the two hard constraints — very low latency, 100% serverless — the pixel stacks all fail for the *interior* loop: they are server-bound, hundreds-of-ms-to-seconds per frame, and lossy/non-deterministic (a guessed parse of pixels can't re-derive, breaking Law L5). The one worth keeping, and only at the **raster boundary** (non-κ content — an external iframe, an uploaded image), is a small **Qwen2.5-VL** as the mux "vision" specialist, whose output is *sealed back as a κ-object* so the interior stays deterministic.

**Decision.** Ship perception as a content-addressed scene over the κ-graph, factored so the cheap/deterministic interior and the heavy/edge vision are cleanly separated:

1. **Two faces per object.** A scene entry pairs a `code` κ (source/state/compute output) with a `visual` κ (rendered fragment). `observeCode`/`observeVisual` merge into the entry; the renderer's `onFinal({id, kappa})` auto-feeds the visual face and the orchestrator's dispatch auto-feeds the code face for the SAME id — so the orchestrator perceives both code and screen for one logical object, in real time.
2. **O(1) deltas + a whole-scene κ.** Every observe is a content-address compare: an unchanged re-observe is a free no-op; a change emits a delta tagged `code | visual | both`. `sceneKappa()` content-addresses the entire scene (cached until dirty), so the orchestrator skips an unchanged pass in O(1). No pixels, no VLM, ever, in this loop — this is the low-latency, serverless sense.
3. **Drift = the real-time feedback signal.** `feedback()` reports, per object, `coherent` (both faces fresh), `codeOnly`/`visualOnly`, and **drift** (the code moved but the screen hasn't caught up — `codeSeq > visualSeq`, deterministic via a monotonic counter, no clocks). Drift is exactly what an automatic self-improvement loop over streaming AI + real-time rendering needs: it sees when an action changed the code but the render lagged or failed, and acts. Because the substrate makes render a deterministic function of code, **coherent is the default** and drift only appears on lag/failure — a precise, honest health signal.
4. **Vision only at the raster edge.** Non-κ content is lifted into structure on demand by the mux "vision" specialist (a small on-device VLM), and its output is sealed as a new κ-object that joins the scene. Vision CREATES κ-structure where there was none; it is never the primary sense and never in the inner loop.

**The self-improvement loop, closed.** observe (scene delta) → evaluate (coherent vs drift, plus the fabric's latency/cache-hit telemetry, ADR-0085, and receipt validity) → act (mux/fabric) → re-observe. The signal feeds the `reevaluate` hook (ADR-0084) and the append-only trace corpus (ADR-0081 evolve), governed by the conscience + succession gate — so the orchestrator learns which specialist/render choices measurably reduced drift and latency, and cannot Goodhart on a self-grade.

**Honest costs.** The scene sees only what is κ-addressed/rendered through the substrate — third-party raw-canvas/iframe content is invisible until the edge VLM lifts it (the one slow, heavy, non-deterministic place pixels enter; keep it on-demand, out of the hot loop). Coverage is as complete as the producers that register: the renderer auto-covers the visual face and the orchestrator the code face, but every widget/app/object must register to be fully seen (wiring, deferred). And "self-improvement" is only as good as its objective — drift + latency + receipt validity are real signals, but a vibe is not.

**Composition.** Driven by 0081 (the orchestrator loop), over 0084 (the specialist fleet + the "vision" task) and 0085 (the fabric telemetry + the renderer's `onFinal`); verifies via 0052/0024/0025 (κ + receipt) and gates via 0033. Mints nothing — the scene is the κ-graph observed. Pending: `tools/holo-perception-witness.mjs` + `#holo-perception`, full producer coverage, the raster-edge VLM specialist, and the re-lock.
