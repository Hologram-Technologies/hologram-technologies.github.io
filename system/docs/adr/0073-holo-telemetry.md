# ADR-0073: Holo Telemetry — system-wide observability native to the UOR substrate. Adopt the OpenTelemetry data model and W3C Trace Context as content-addressed UOR objects (reject the Prometheus scraper): a span is a PROV-O Activity whose W3C ids are derived from the operation's content, so a signal's identity re-derives by holding only its κ; wall-clock numbers are honestly host-attested, never claimed as re-derived

**Status:** Proposed — witnessed by `tools/holo-telemetry-witness.mjs` (pure Node, joining the gate's LIVE set) and a required `#holo-telemetry` row in `os/etc/conformance.jsonld` (the gate, ADR-024). The runtime ships at `os/usr/lib/holo/holo-telemetry.mjs` (`window.HoloTelemetry`), is sealed as a self-verifying UOR object by `tools/seal-telemetry.mjs` (→ `os/usr/share/ns/telemetry.jsonld` (the `hostel:` ontology) + `os/etc/holo-telemetry/telemetry.uor.json`), is surfaced to every builder through the SDK façade (`os/usr/lib/holo/holo-sdk.js` — new verbs `telemetry`/`trace`/`meter`/`verifyTelemetry`), and is auto-wired **system-wide** by the theme runtime (`os/usr/lib/holo/holo-theme.js`, `bootHoloTelemetry` — the same one-canonical-wire as Holo UX/FX, Law L2, no per-app tag). Runs identically in the browser and in Node. The decision is not done until it is witnessed (ADR-024); this ADR pins the design and carries the build spec. This delivers the "OpenTelemetry-aligned observability (Stage 3)" already promised by Holo SDK (ADR-0050). *(House-keeping: numbered `0073` — re-grep `docs/adr/` for the max before adding the next, since numbers advance concurrently.)* Composes Holo Orchestrate (ADR-0045, the PROV-O work receipt a trace specializes), Holo Route (ADR-0069), the Holo Constitution (ADR-0033, the egress gate), and Own/Pin (ADR-0053, the exporter). Mints one namespace, `hostel:`.

**Context.** OS2 had no observability layer. A system you cannot observe you cannot operate, debug, or trust — yet the two dominant ways to add observability both fight the substrate's grain. **Prometheus** is a *server*: a long-running process that pull-scrapes `/metrics` HTTP endpoints on a timer, stores the samples in its own time-series database, and queries them with PromQL. Its whole architecture assumes the things the substrate rejects — a central authority, mutable append-only storage, and, fatally, *trust the emitter*: a target reports `request_duration_seconds 0.05` and Prometheus believes it. Running it would mean running a foreign runtime, which the standing strategy forbids (ADR-0029, "adopt OSS standards as κ-objects, never run a foreign runtime"). **OpenTelemetry** is the opposite kind of thing: not a server but a *specification* — a vendor-neutral data model (the three signals: traces/spans, metrics, logs, over a Resource and an instrumentation Scope), a set of semantic conventions, and a wire format (OTLP). Crucially, its trace-propagation format *is* **W3C Trace Context** (`traceparent`), a ratified W3C Recommendation — which is exactly the open-semantic-web alignment this work requires. So the data model is adoptable; the scraper is not.

The deeper point is that the substrate already *is* most of an observability backend, it just was not named as one. A **trace** — a tree of timed operations with causal parent links — is precisely a **PROV-O execution DAG**, which Holo Orchestrate (ADR-0045) already mints as a verifiable *work receipt*. The **collector** that OTel ships to is the **κ-store** (already here). The **query** layer is **graph traversal** (Holo Resolve, ADR-0046). The **exporter** is **Pin/Own** (ADR-0053). The **privacy boundary** OTel leaves to configuration is the **conscience gate** (ADR-0033). Nothing new needs to be built; the signals need to be *named as substrate objects*.

And naming them that way fixes the one thing OTel inherits from every telemetry system before it: it is trust-the-emitter. A span *claims* a duration and a parent; you believe both. On a substrate whose first law is *identity is content* and whose discipline is *verify, don't trust*, a span can do better — its W3C trace-id and span-id can be **derived from the content of the operation** rather than drawn from a random source, so the identity of a signal **re-derives** (Law L5) and a tampered structural field breaks it. That is the gap, and the move.

**Decision.** Make every telemetry signal a content-addressed UOR object, and be honest about which of its facts re-derive:

1. **Adopt the OpenTelemetry data model + W3C Trace Context; reject the Prometheus scraper.** The vocabulary is OTel's — `Span · Metric · LogRecord · Resource · Scope` — minted as the `hostel:` ontology where each term declares `skos:closeMatch` to its OTel/W3C authority (the semantic-web bridge), and propagation is the verbatim W3C `traceparent` form `version-traceid-spanid-flags` (16-byte trace-id, 8-byte span-id; the all-zero id refused per the Recommendation). No central server, no pull-scrape, no PromQL — the κ-store is the collector, graph traversal the query, Pin/Own the exporter.

2. **A span IS a PROV-O Activity; its W3C ids are derived from its content.** A span's *structural core* — name · kind · attributes · the operation's `prov:used`/`prov:generated` κ · its parent span-id — is canonicalized (JCS) and hashed; the **span-id is the low 8 bytes of that hash, the trace-id the low 16 of the root span's**. So a W3C-conformant 64/128-bit id is *simultaneously* a UOR content address. Holding only the span's κ, a peer re-derives the span-id from the stored structural fields and compares (Law L5) — tamper the name, the attributes, the parent, or the i/o κ and the id no longer re-derives, so the span is refused. A **trace** is the DAG of spans sharing one trace-id, each `prov:wasInformedBy` its parent — the OTel trace expressed as the same self-verifying PROV-O receipt Holo Orchestrate already proves.

3. **The honest split — structural facts re-derive, wall-clock facts are attested.** This is the design, not a caveat. A span's *identity and provenance* (which operation, its inputs→outputs, its parent, its status, the conscience verdict at dispatch) are structural and **re-derive** (`hostel:rederivable: true`). A span's *timing* (start, end, duration) and a metric's *measured value* are wall-clock observations that **cannot** re-derive — so they are carried in a separate `hostel:measurement` block marked `hostel:rederivable: false` and **signed by the local host key** (an attested claim, exactly what a latency actually is). Telemetry never claims to verify a number it cannot recompute; it states, per fact, whether the fact re-derives. A tampered structural byte breaks the id; a tampered measurement breaks the κ (and the host signature) — both are refused, by the appropriate mechanism.

4. **No new infrastructure (NIHITO).** Metrics (`sum · gauge · histogram`) and log records seal to κ-objects the same way: a structural identity (name·unit·kind·labels for a metric; severity·body·correlation-ids for a log) that re-derives, plus an attested observation. Every signal lands in the κ-store; querying a trace is graph traversal; persisting one durably is Pin/Own; correlating a log to a span is the shared W3C trace-id/span-id. The OTLP/JSON envelope is emitted on demand so any off-substrate OpenTelemetry collector can ingest the data — interop without adoption of its trust model.

5. **Private-first egress, fail-closed (Law L1).** Telemetry is **local-only by default** — the substrate observes itself without anything leaving the device. Export is a *decision*, default-deny: `exportTo` refuses without explicit consent, and even with consent it passes the conscience gate (ADR-0033) before a byte leaves; a blocking verdict refuses the export. There is no silent phone-home, no ambient collection endpoint. This is the inversion of the Prometheus model, where being scraped is the default.

6. **System-wide by one canonical wire (Law L2).** The runtime self-registers `window.HoloTelemetry` on `holo-app-ready`; the theme runtime — which every Holo Theme/UI citizen already loads — boots it with no per-app script tag, exactly as it boots Holo UX and Holo FX. A tiny `PerformanceObserver` bridge turns the `performance.measure` marks apps *already emit* into local spans, so the system is instrumented system-wide with zero per-app code. Apps that want explicit spans/metrics call the SDK verbs; everything else is observed for free, locally.

**Magical for builders — the seamless surface.** One import, the same mental model as OpenTelemetry, but every signal is a verifiable object and nothing leaves the device unasked:

```js
import { trace, meter, verifyTelemetry } from "@hologram/sdk";

const tr   = await trace("checkout", "1.0");
const span = tr.startSpan("charge", { attributes: { amount: 42 }, used: orderκ, generated: receiptκ });
// … do the work …
await span.end({ status: "ok" });          // timing is host-attested; the span-id is content-derived
const { kappa } = await tr.seal();         // the whole trace is one self-verifying PROV-O κ

await verifyTelemetry(kappa);              // re-derive every span-id from content (verify, don't trust)
```

An **agent** gets the same over MCP-shaped verbs (the SDK façade): start a span by content, seal a trace to a κ, verify any signal — and that trace κ slots straight into an Orchestrate work receipt as a deterministic activity, because it already *is* one.

**Relationship to Orchestrate and Route (no overlap).** **Holo Orchestrate** (ADR-0045) is the *collaboration* receipt — *who* acted (agents, with identity, reputation, the UCAN they held). **Holo Route** (ADR-0069) is the *data* boundary — typed κ-transforms whose seams re-derive. **Holo Telemetry** is the *observation* plane — the running record of *what happened, when, how long*, across both. A trace is the deterministic, time-stamped sibling of an Orchestrate DAG: where Orchestrate proves a collaboration and Route proves a composition, Telemetry observes either and seals the observation as one more verifiable object. It re-uses Orchestrate's PROV-O Activity shape and never re-implements it.

**Consequences.** OS2 becomes observable on its own terms: every span, metric, and log is a `holo://κ` you can verify, share, own, pay against, or delegate — not a sample in someone's central TSDB. Three failures that classical telemetry punts on dissolve. **Forged signals** (a span re-derives from the operation's content, so a fabricated or altered span is refused) — the un-gameable-telemetry property Holo AgentTrust (ADR-0039) reached for, now ambient. **Dishonest precision** (the system never claims to verify a wall-clock number; it marks exactly which facts re-derive and which are host-attested). **Silent exfiltration** (local-only by default, conscience-gated egress, no ambient endpoint). The honest scope: the host-key *attestation* of wall-clock measurements is only as strong as the local identity key (real EdDSA in the browser via the self-sovereign identity; the witness uses a deterministic stub) — it proves *this host claimed this number*, not that the number is physically true, which is the most any telemetry can honestly assert. Wall-clock values therefore do not re-derive and are not meant to; the re-derivable surface is identity and provenance, which is where trust actually matters. It composes the catalog rather than extending it — Orchestrate's receipt, the κ-store, graph traversal, Pin/Own, the conscience gate — and mints exactly one new thing the substrate lacked: the observability plane, native.

**External authorities.** The OpenTelemetry data model + OTLP (the signal model and wire format adopted) · W3C Trace Context (`traceparent`, the propagation format) · W3C PROV-O (a span is a `prov:Activity`; a trace is a PROV-O DAG) + DID Core (`did:holo`) + Subresource Integrity / VC Data Integrity (the digests) · IETF RFC 8785 (JCS, the canonical form every signal id derives from) · UOR-ADDR (κ-label = H(canonical_form); the W3C id is the low bytes of a content address) · holospaces Laws L1 (identity is content; private-first, local-only by default) / L2 (one canonical wire — the theme boots telemetry system-wide, no per-app tag) / L5 (a signal's identity is accepted only after re-deriving it) · the Holo Constitution (ADR-0033, the egress conscience gate, fail-closed). Explicitly **rejects** Prometheus's central pull-scrape server architecture (trust-the-emitter, a foreign runtime). Mints one namespace, `hostel:`.
