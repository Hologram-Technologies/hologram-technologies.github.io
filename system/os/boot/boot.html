About Holo Boot

Holo Boot 0.1 · rEFInd-faithful · Hologram OS

Holo Boot is the hologram-native rEFInd: a graphical boot manager that runs entirely in the browser, with no server and no install. Where rEFInd scans the EFI System Partition for OS loaders and boots them, Holo Boot scans the κ-store for holospaces and boots them — each identified by its content hash, not its location (Law L1).

Secure Boot = κ re-derivation (Law L5)

Before a loader boots, Holo Boot fetches its bytes and re-derives their κ, refusing any entry whose content does not match the κ pinned in boot/boot-manifest.json. This is the content-addressed analog of Secure Boot / shim signature verification — a tampered loader is rejected.
